## 設計判断（サンプル）

以下はテンプレート採用時点の判断をサンプルとして残したもの。実プロジェクトの判断に置き換えてよい。

### なぜ Hono か

- Cloudflare Workers のランタイム（V8 isolate、Node.js API 非互換）で軽量に動くルーターが必要だった
- Express 系はミドルウェアエコシステムが Node.js 前提で Workers 環境と相性が悪い
- Hono は Workers/Deno/Bun 等の edge ランタイムをファーストクラスでサポートし、型安全なルーティング（`hono/jsx` を含む）を提供する
- トレードオフ: エコシステムの成熟度は Express に劣るが、edge 実行という制約下では優位性が上回る

### なぜ D1 か

- Cloudflare が提供するマネージド SQLite 互換 DB で、Worker と同一プラットフォーム内でレイテンシが低い
- リレーショナルなクエリ（JOIN・トランザクション境界）が必要な要件があり、KV のようなキーバリューストアでは表現力が不足する
- トレードオフ: 外部マネージド DB（PlanetScale 等）に比べるとスケール上限や機能（フルテキスト検索等）に制約があるが、初期構成のシンプルさと運用コストの低さを優先した
