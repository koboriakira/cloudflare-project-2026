# test-and-fix — テスト実行と修正フロー

失敗しているテスト・型エラー・lint 違反を洗い出し、修正するときに使う。

## 手順

1. `npm run typecheck` を実行し、型エラーを確認する
2. `npm test` を実行し、失敗しているユニットテストを確認する
3. `npm run lint` を実行し、スタイル違反を確認する
4. それぞれの失敗について、実装側の問題かテスト側の問題かを切り分けてから修正する
   - 実装がテストの意図する仕様を満たしていない → 実装を直す
   - テストが古い仕様を前提にしている → テストを更新する（理由をコメントせず、コミットメッセージで説明する）
5. 修正後、`npm run typecheck` → `npm test` → `npm run lint` を再実行し、すべて成功することを確認する
6. E2E に影響する変更（ルーティング・レスポンス形式）があれば `npm run test:e2e` も実行する

## 注意

- 3回連続で同じアプローチが失敗したら、いったん止まって方針を再検討する
- lint は `npm run lint:fix` で自動修正できるものから試す
