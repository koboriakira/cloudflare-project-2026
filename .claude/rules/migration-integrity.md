# マイグレーション連番整合性

## 守るべき不変条件

`migrations/` 以下の SQL ファイルについて、以下が常に真であること。

1. **ファイル名は `NNNN_description.sql` 形式** — `NNNN` は4桁ゼロパディングの連番、`description` は小文字スネークケース
2. **連番は `0001` から始まり途切れない** — 欠番があってはならない
3. **既存のマイグレーションファイルは編集しない** — スキーマ変更が必要な場合、既存ファイルを書き換えず新しい連番のファイルを追加して対応する

## なぜこのルールが必要か

- D1（SQLite 互換）のマイグレーションは適用順に依存する。連番が途切れたり重複したりすると、環境によって適用順序が変わり、開発環境と本番環境でスキーマが乖離する
- 一度適用されたマイグレーションファイルを事後編集すると、「そのファイルを実行済みの環境」と「未実行の環境」でスキーマが分岐する。ロールバック・再適用の追跡が不可能になる
- ファイル名パターンを固定することで、D1 の適用順とファイルシステム上のソート順を一致させ、CI での機械検証を可能にする

## 違反時の対応

`judges/migration-integrity.sh` で検出する。
